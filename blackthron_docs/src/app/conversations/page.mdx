# Batch Analysis

Submit up to 25 transactions in a single API call. Each transaction is analyzed independently in parallel — the response contains an ordered array of analysis results matching the input order. {{ className: 'lead' }}

---

## Analyze a batch {{ tag: 'POST', label: '/v1/analyze/batch', id: 'analyze-a-batch' }}

<Row>
  <Col>

    ### Required attributes

    <Properties>
      <Property name="transactions" type="array">
        Array of transaction objects. Maximum 25 items per request.
      </Property>
      <Property name="transactions[].transaction" type="string">
        Base64-encoded serialized Solana `VersionedTransaction`.
      </Property>
      <Property name="transactions[].cluster" type="string">
        Solana cluster: `mainnet-beta` or `devnet`.
      </Property>
    </Properties>

    ### Optional attributes

    <Properties>
      <Property name="policy" type="object">
        Policy rules applied to all transactions in the batch. Can be overridden per transaction.
      </Property>
      <Property name="policyProfile" type="string">
        Pre-configured policy profile for all transactions: `strict`, `defi-permissive`, or `monitor-only`.
      </Property>
    </Properties>

    The response `results` array preserves the same order as the input `transactions` array. Each element is a full [analysis object](/contacts#the-analysis-object).

    A batch request counts as one API call regardless of the number of transactions included.

  </Col>
  <Col sticky>
