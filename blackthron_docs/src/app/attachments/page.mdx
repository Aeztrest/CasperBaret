# Replay

The replay endpoint re-runs a transaction simulation at a specific historical slot. Use it to audit past analysis decisions, reproduce incidents, or test how your policy configuration would have evaluated historical transactions. {{ className: 'lead' }}

---

## Replay a transaction {{ tag: 'POST', label: '/v1/replay', id: 'replay-a-transaction' }}

<Row>
  <Col>

    ### Required attributes

    <Properties>
      <Property name="transaction" type="string">
        Base64-encoded serialized Solana `VersionedTransaction`.
      </Property>
      <Property name="cluster" type="string">
        Solana cluster: `mainnet-beta` or `devnet`.
      </Property>
      <Property name="slot" type="number">
        The slot number at which to replay the simulation. Must be recent enough that the RPC node still has state for it (typically within the last few epochs).
      </Property>
    </Properties>

    ### Optional attributes

    <Properties>
      <Property name="policy" type="object">
        Inline policy rules to evaluate against the replayed analysis. Useful for testing what a new policy configuration would have done.
      </Property>
      <Property name="policyProfile" type="string">
        Pre-configured policy profile: `strict`, `defi-permissive`, or `monitor-only`.
      </Property>
    </Properties>

    The response structure is identical to a standard `/v1/analyze` response, with an additional `slot` field indicating which slot was used for simulation.

    <Note>
      Replay uses the Solana RPC `simulateTransaction` with the `minContextSlot`
      parameter set to your requested slot. If the RPC node no longer has state
      for that slot, the request will fail with a `SLOT_NOT_AVAILABLE` error.
    </Note>

  </Col>
